## Known caveats

- Schema validation remains distinct from semantic conformance. In
  particular, the published schema does not require the root `LoTE` property
  and misplaces one `additionalProperties: false` entry inside
  `ServiceDigitalIdentity.properties`; local semantic checks must cover these
  published-schema gaps without modifying the pinned source.
- Compact JAdES Baseline B validation remains unsupported, so a schema-valid
  JSON LoTE still cannot receive a complete conformance verdict.
- Exact clause 6.1 lexical/URI/language/country rules and calendar-month date
  semantics remain TS602-06/TS602-07 work.
- The compatibility adapter intentionally recognizes only the documented
  `LoTE.TrustedEntitiesList.TrustServiceProvider` legacy path.
- `npm install` reported five repository dependency audit findings (three
  moderate, one high, one critical); no breaking `npm audit fix --force` was
  applied because dependency remediation is outside this task.
