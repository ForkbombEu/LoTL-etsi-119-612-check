## Implementation Notes

- Canonical namespace `http://uri.etsi.org/02231/v2#` together with
  `TSLVersionIdentifier=6` now selects the integrity-checked V2.4.1 base XSD
  and offline catalog automatically.
- The selected finding records the normative document/Annex C citation,
  official attachment identity, immutable Forge tag/commit, schema/catalog
  hashes and all bundle-integrity failures if present.
- A canonical namespace with another or missing format version is
  `inconclusive`; the tool does not apply V2.4.1 to ambiguous evidence.
- The observed `http://uri.etsi.org/19612/v2.4.1#` EUDI RI namespace remains
  an explicit compatibility input with `normativeStatus: not_established`.
  It receives an `inconclusive` schema finding without invoking `xmllint`.
- CLI `--xsd` takes precedence over automatic selection, records the
  override path/hash/size, and retains target-namespace matching and
  `xmllint --nonet` behavior. API requests intentionally do not accept local
  filesystem overrides.
- Structured diagnostics can now identify `artifact.xml` or an exact pinned
  schema/dependency path while temporary directories remain redacted.
- Pinned validation reports `unsupported` when integrity fails, the bundle
  cannot load or `xmllint` is absent. Unsupported/ambiguous selection never
  pretends schema validation passed.
- The `ts119612.binding.xml_schema` ledger family is now implemented. Ledger
  coverage is 69 total: 2 implemented, 29 partial and 38 not implemented.

## Known Caveats

- No authoritative schema has been identified for the observed EUDI RI
  compatibility namespace. A human may supply a matching explicit CLI
  override, but the tool will not infer one.
- Only format version 6/V2.4.1 is pinned. Other canonical namespace format
  versions remain inconclusive rather than being validated with the wrong XSD.
- The existing `tsl-valid-ish.xml` fixture models the compatibility namespace
  and is not structurally valid against the official canonical schema; mocked
  command runners keep normal routing tests deterministic and offline.
- Schema validity remains separate evidence. Annex C gives the normative
  document precedence over conflicts in the electronic schema.
- Complete scheme-information semantics remain TS612-04 work.
