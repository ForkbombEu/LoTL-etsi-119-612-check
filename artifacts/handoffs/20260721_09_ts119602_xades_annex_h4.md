## Implementation Notes

- Added a reusable XAdES assessor for direct `QualifyingProperties` incorporation, signed `SignedProperties`, Baseline B `KeyInfo/X509Data`, reference and transform cardinalities, supported canonicalization, UTC `SigningTime`, `SigningCertificateV2`, certificate-digest matching, `DataObjectFormat`, reference digest structure, and prohibited legacy properties.
- Pub-EAA XML LoTEs receive Annex H.4 findings for the enveloped signature, empty-URI document reference, exact two-transform order, and exclusive canonicalization. Other XML signature profiles receive `not_applicable` for those profile-only checks.
- The existing xmlsec result remains the cryptographic result for all references, digests, and the signature value; the structural findings do not substitute for cryptographic verification.
- The embedded signing certificate is parsed into the existing evidence model. Its country and organization are compared exactly with `SchemeTerritory` and all `SchemeOperatorName` values.
- Certificate validity at assessment time and signer trust are separate stable findings. Trust remains `not_checked` unless an explicit trusted signer fingerprint set is supplied; embedded certificate material is never treated as trusted by itself.
- Updated the requirements ledger and coverage counts. The common Baseline B family remains partial until JAdES is implemented, while the Pub-EAA signature family records the implemented XML constraints.

## Known Caveats

- The focused XAdES fixture is a deterministic structure template hydrated in tests with the existing canonical signing certificate. Its digest/signature values are placeholders, so cryptographic outcomes are supplied by the unit-test verifier rather than presented as a genuinely signed artifact.
- Production XML LoTE assessment currently supplies no trusted signer fingerprint set, so signer trust is intentionally `not_checked`; certificate-chain/path validation against a configured trust source remains contextual backlog work.
- Certificate validity is reported at assessment time. Revocation status and historical validity at a validated signing time are not inferred.
- Cryptographic verification still depends on the bounded `xmlsec1` adapter and reports `not_checked` when that backend is unavailable.
- XML XSD validation is a separate schema backlog item and is not conflated with the signature findings added here.
