## Implementation Notes

- Added strict compact-JWS detection and parsing with canonical base64url segments, protected-header parsing, attached JSON payload recovery, preservation of the original signing input, and signed-payload comparison.
- Added stable findings for JAdES serialization, protected Baseline B properties, critical headers, claimed signing time, certificate references, payload recovery/matching, algorithm support, verification attempt/result, certificate parsing/validity, signer subject matching, and explicit trust.
- Implemented local verification for RSA PKCS#1, RSA-PSS, ECDSA, and EdDSA JWS algorithm families using the embedded protected `x5c` signing certificate. Unsupported algorithms are reported as `unsupported` without claiming a verification attempt.
- Enforced the post-transition protected `iat` form used by the selected JAdES V1.2.1 profile, prohibited `sigT` and SHA-1 `x5t`, validated `x5t#S256`, `x5t#o`, and `sigX5ts` syntax, and required critical extension declarations where applicable.
- Raw JSON remains assessable but now fails the mandatory compact-signature requirement. A legacy JSON `signature` object remains non-normative evidence and cannot satisfy JAdES checks.
- Routed recovered JWS payloads through the existing JSON schema and semantic assessor, added `jws` to report/OpenAPI format enums and fetched-artifact extensions, and exposed signer certificates through the existing report model.
- Updated requirements-ledger coverage and marked TS602-10 complete, with TS602-11 next.

## Known Caveats

- Detached compact-JWS payloads are detected but remain `unsupported` because this assessment path does not accept separate exact payload bytes.
- External certificate resolution for `x5t#S256`, `x5t#o`, or `sigX5ts` without protected `x5c` material is not implemented.
- Embedded `x5c` certificates are parsed as evidence, but chain/path construction, revocation, and production trust-source integration remain contextual work. Production audit paths therefore leave signer trust `not_checked` unless a caller invokes the core assessor with an explicit fingerprint set.
- The committed signing certificate is self-signed test material and is trusted only in the focused explicit-fingerprint unit test. Its temporary private key and PEM generation files were deleted and are not committed.
- Complete Annex D-I dispatch and profile-specific constraints remain incomplete, so successful schema and JAdES checks cannot produce a complete TS 119 602 conformance verdict.
