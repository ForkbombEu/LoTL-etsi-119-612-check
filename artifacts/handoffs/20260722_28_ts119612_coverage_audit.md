## Implementation Summary

- Added a structured per-artifact audit containing all 69 TS 119 612 ledger
  families, including non-applicable TL-only families for LoTL artifacts.
- Each family records applicability, implementation state, evidence scope,
  citations, linked check IDs, observed finding statuses, outcome and whether
  the evidence is conclusive.
- Added explicit blocker lists for partial, not-implemented, contextual and
  non-conclusive implemented families.
- Replaced the hard-coded incomplete-coverage argument with
  `completeVerdictEligible`, which requires at least one applicable family,
  complete implementation of every applicable family and conclusive evidence
  from every applicable implemented family.
- Contextual assessment now rebuilds the coverage audit after replacing
  contextual findings, preventing stale pre-context results.
- Complete failures remain conclusive evidence but are handled independently
  by the conformance verdict. Unsupported, inconclusive and not-checked results
  cannot yield a complete verdict.
- Bumped the stable report contract to schema v6, exposed the audit in
  OpenAPI, rendered the same 69-family table in Markdown and added missing TS
  119 612 unsupported/inconclusive report-summary counters.
- Added a manual, bounded EUDI RI/WE BUILD live-smoke runbook. No live network
  activity is part of the normal test suite.

## Known Caveats

- The current ledger remains 15 implemented, 45 partial and 9 not implemented;
  complete TS 119 612 verdicts therefore remain disabled by design.
- Coverage associates runtime findings through the ledger's maintained
  `existingCheckIds`; newly added findings must keep that mapping current.
- Live reference endpoints can drift. The manual procedure records drift as
  evidence and does not infer normative rules from one response.
- Direct API smoke results contain fetch/report evidence but do not persist the
  fetched XML response body.
- External `xmllint` and `xmlsec1` availability still controls whether their
  respective runtime checks are conclusive.
